# Minerva 시스템 테스트 가이드

## 개요
이 문서는 Minerva 학습 메이트 시스템의 정상 작동 여부를 검증하기 위한 테스트 가이드입니다.

## 시스템 구성 요소 테스트

### 1. Sisyphus 테스트
**테스트 내용**: 
- 사용자 요청을 정확히 분석하고 작업 유형을 판단하는지 확인
- 단순/복잡한 작업에 따라 적절한 에이전트를 호출하는지 검증
- taskcreate 명령어가 제대로 작동하는지 확인

**예상 결과**: 
- 작업 유형이 정확히 분류됨
- 적절한 에이전트가 호출됨

### 2. Prometheus 테스트
**테스트 내용**:
- 복잡한 작업의 구조를 정확히 설계하는지 확인
- 학습 계획을 마크다운 형식으로 생성하는지 검증
- 필요 기술 스택 및 의존성 확인이 정확한지 확인

**예상 결과**:
- 작업 내용이 적절히 분석됨
- 학습 계획이 정확하게 생성됨

### 3. Atlas 테스트
**테스트 내용**:
- Prometheus의 계획을 구체화하고 실행하는지 확인
- 병렬 작업(Explore, Librarian)이 정상적으로 작동하는지 검증
- 오류 처리 로직이 작동하는지 확인

**예상 결과**:
- 계획 수립 → 검증 → 구현 순서로 작동
- 실패 시 재시도 기능 작동

### 4. 워크플로우 테스트
**테스트 내용**:
- Learn, Done, Hint, Review, Fix, List 워크플로우가 각각 정확히 작동하는지 확인

**예상 결과**:
- TDD 방식으로 학습 프로세스 진행
- 각 단계별 기능이 정확하게 수행됨

### 5. 스킬 테스트
**테스트 내용**:
- Builtin, Dynamic, Internal 스킬들이 정확히 작동하는지 검증

**예상 결과**:
- 스킬별 기능이 적절히 적용됨
- 에이전트 호출 시 적절한 스킬이 로드됨

## 테스트 절차

### 1. 기본 테스트
1. Sisyphus가 작업 유형을 정확히 판단하는지 확인
2. 작업 생성 및 Atlas 실행 확인
3. 워크플로우 호출 확인

### 2. 상세 기능 테스트
1. TDD 방식으로 코드 구현 테스트
2. 힌트 제공 기능 테스트
3. 코드 검토 기능 테스트
4. 오류 수정 기능 테스트

### 3. 시스템 통합 테스트
1. 모든 구성 요소가 정상적으로 상호작용하는지 확인
2. 메모리 관리 및 컨텍스트 주입 검증
3. 에이전트 간 협력 기능 확인

## 예상 결과 검증 기준

### 1. 정확성
- 작업 유형 판단 정확도: 95% 이상
- 에이전트 호출 정확도: 90% 이상
- 학습 계획 생성 정확도: 90% 이상

### 2. 효율성
- 작업 처리 시간: 평균 30초 이내
- 메모리 사용량: 최대 50MB 이하
- 에이전트 호출 응답 시간: 평균 5초 이내

### 3. 안정성
- 시스템 다운 시간: 연간 1시간 이하
- 오류 발생률: 0.1% 이하
- 자동 복구 기능 작동률: 99% 이상

## 문제 해결 절차

### 1. 테스트 실패 시 절차
1. 로그 파일 확인 (logs/ 폴더)
2. 작업 상태 확인 (.sisyphus/tasks.json)
3. 에이전트 호출 로그 검토
4. 스킬 로드 확인

### 2. 일반적인 문제 해결
- 시스템 재시작: `systemctl restart minerva`
- 캐시 정리: `rm -rf .cache/minerva/`
- 설정 파일 재로드: `minerva reload-config`
